title: Video Interface

The Video interface provides mid-level abstraction for capturing images from video devices such as webcams. It handles platform-specific video APIs including V4L2 (Linux), Video for Windows (VfW), and DirectShow (Windows).

## Overview

The video interface:
- Opens and probes video devices
- Negotiates image formats
- Captures video frames as zbar_image_t objects
- Provides device control management (brightness, contrast, etc.)
- Supports multiple I/O modes (read, mmap, USERPTR)

## Basic Usage

```c
zbar_video_t *video = zbar_video_create();
zbar_video_open(video, "/dev/video0");
zbar_video_request_size(video, 640, 480);
zbar_video_init(video, zbar_fourcc('Y','8','0','0'));
zbar_video_enable(video, 1);

zbar_image_t *image = zbar_video_next_image(video);
// ... process image ...

zbar_video_enable(video, 0);
zbar_video_destroy(video);
```

## Type Definitions

### zbar_video_t

Opaque video object that manages video device access and frame capture.

### video_control_type_t

Enumerates video control types:

| Value | Description |
|-------|-------------|
| `VIDEO_CNTL_INTEGER` | Integer-valued control |
| `VIDEO_CNTL_MENU` | Menu selection control |
| `VIDEO_CNTL_BUTTON` | Button/action control |
| `VIDEO_CNTL_INTEGER64` | 64-bit integer control |
| `VIDEO_CNTL_STRING` | String-valued control |
| `VIDEO_CNTL_BOOLEAN` | Boolean on/off control |

### video_controls_t

Structure representing a video device control:

```c
typedef struct video_controls_s {
    char *name;                    // Control name
    char *group;                   // Control group/class
    video_control_type_t type;     // Control type
    int64_t min, max, def;        // Min, max, default values
    uint64_t step;                 // Increment step
    unsigned int menu_size;        // Size of menu array
    video_control_menu_t *menu;    // Menu items (for menu type)
    void *next;                    // Next control in list
} video_controls_t;
```

**Since:** 0.20

### video_resolution_s

Structure representing an available video resolution:

```c
struct video_resolution_s {
    unsigned int width;   // Width in pixels
    unsigned int height;  // Height in pixels
    float max_fps;        // Maximum frame rate
};
```

**Since:** 0.22

### zbar_video_open

```c
int zbar_video_open(zbar_video_t *video, const char *device)
```

Open and probe a video device.

<ParamField path="video" type="zbar_video_t*">
  Video object
</ParamField>

<ParamField path="device" type="const char*">

  - Linux: V4L2 device node path (e.g., "/dev/video0")

</Note>

### zbar_video_get_fd

```c
int zbar_video_get_fd(const zbar_video_t *video)
```

Retrieve file descriptor for opened video device. Useful for `select()`/`poll()` to detect when new frames are available.

<ParamField path="video" type="const zbar_video_t*">
  Video object
</ParamField>

**Returns:** File descriptor, or -1 if device is not open or driver only supports V4L1

<Warning>
Only works with V4L2 drivers on Linux. Not supported on Windows or V4L1 drivers.
</Warning>

### zbar_video_request_iomode

```c
int zbar_video_request_iomode(zbar_video_t *video, int iomode)
```

Request preferred I/O mode for debug/testing. Errors occur if the driver does not support the specified mode.

<ParamField path="video" type="zbar_video_t*">
  Video object
</ParamField>

<ParamField path="iomode" type="int">
  I/O mode:
  - `0` - Auto-detect (default)
  - `1` - Force I/O using `read()`
  - `2` - Force memory-mapped I/O using `mmap()`
  - `3` - Force USERPTR I/O (V4L2 only)
</ParamField>

**Returns:** 0 on success, -1 if device is already opened or invalid mode

<Note>
Must be called before `zbar_video_open()`.
</Note>

**Since:** 0.7

### zbar_negotiate_format

```c
int zbar_negotiate_format(zbar_video_t *video, zbar_window_t *window)
```

Select a compatible format between video input and output window. The algorithm attempts to:
1. Use a format shared by video input and window output
2. Select a format most useful for barcode scanning
3. Minimize format conversion cost if necessary

<ParamField path="video" type="zbar_video_t*">
  Video object (must be opened)
</ParamField>

<ParamField path="window" type="zbar_window_t*">
  Window object, or NULL to negotiate format without window constraints
</ParamField>

**Returns:** 0 on success, -1 on error

**Example:**
```c
zbar_video_t *video = zbar_video_create();
zbar_window_t *window = zbar_window_create();

zbar_video_open(video, "/dev/video0");
zbar_window_attach(window, display, drawable);

// Negotiate best format for both video and window
zbar_negotiate_format(video, window);

zbar_video_enable(video, 1);
```

## Platform-Specific Notes

### Linux (V4L2)

- ZBar supports Video4Linux2 (V4L2) API
- V4L1 is legacy and has limited support
- Requires kernel 3.16 or higher for full feature support
- Device paths typically: `/dev/video0`, `/dev/video1`, etc.
- Most modern webcams and capture devices are supported

**V4L2 I/O Modes:**
- **read()**: Simple but potentially slower
- **mmap**: Memory-mapped buffers (most common)
- **USERPTR**: User-allocated buffers (advanced)

### Windows

**Video for Windows (VfW):**
- Legacy API, limited modern device support
- Simpler but less flexible

**DirectShow:**
- Modern Windows video capture API
- Better device support and performance
- Device paths use DirectShow DevicePath property

**Build Configuration:**
ZBar can be built with either VfW or DirectShow support. Check the binary configuration to determine which is available.

## Common Patterns

### Enumerating Controls

```c
void print_video_controls(zbar_video_t *video) {
    int i = 0;
    video_controls_t *ctrl;
    
    printf("Available video controls:\n");
    while ((ctrl = zbar_video_get_controls(video, i++)) != NULL) {
        printf("  %s", ctrl->name);
        if (ctrl->group)
            printf(" [%s]", ctrl->group);
        printf("\n");
        
        switch (ctrl->type) {
            case VIDEO_CNTL_INTEGER:
            case VIDEO_CNTL_INTEGER64:
                printf("    Type: Integer\n");
                printf("    Range: %ld to %ld (step: %lu)\n",
                       ctrl->min, ctrl->max, ctrl->step);
                printf("    Default: %ld\n", ctrl->def);
                break;
            case VIDEO_CNTL_MENU:
                printf("    Type: Menu\n");
                for (int j = 0; j < ctrl->menu_size; j++) {
                    printf("      [%ld] %s\n",
                           ctrl->menu[j].value,
                           ctrl->menu[j].name);
                }
                break;
            case VIDEO_CNTL_BOOLEAN:
                printf("    Type: Boolean\n");
                break;
        }
    }
}
```

### Video Capture Loop

```c
zbar_video_t *video = zbar_video_create();
zbar_image_scanner_t *scanner = zbar_image_scanner_create();

// Open and initialize
zbar_video_open(video, "/dev/video0");
zbar_negotiate_format(video, NULL);
zbar_video_enable(video, 1);

// Capture loop
while (running) {
    zbar_image_t *image = zbar_video_next_image(video);
    if (image) {
        // Scan for barcodes
        zbar_scan_image(scanner, image);
        
        // Process results...
        
        zbar_image_destroy(image);
    }
}

// Cleanup
zbar_video_enable(video, 0);
zbar_video_destroy(video);
zbar_image_scanner_destroy(scanner);
```

### Using poll() for Non-blocking Capture

```c
#include <poll.h>

zbar_video_t *video = zbar_video_create();
zbar_video_open(video, "/dev/video0");
zbar_negotiate_format(video, NULL);
zbar_video_enable(video, 1);

int fd = zbar_video_get_fd(video);
if (fd >= 0) {
    struct pollfd pfd = {
        .fd = fd,
        .events = POLLIN
    };
    
    while (running) {
        int ret = poll(&pfd, 1, 1000);  // 1 second timeout
        
        if (ret > 0 && (pfd.revents & POLLIN)) {
            zbar_image_t *image = zbar_video_next_image(video);
            // Process image...
            zbar_image_destroy(image);
        }
    }
}
```
