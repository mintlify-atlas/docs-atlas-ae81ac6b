ZBar provides high-level APIs for scanning barcodes from video streams in real-time. This guide covers the Processor API for threaded video scanning and the zbarcam command-line tool.

## Using the Processor API

The Processor is ZBar's highest-level interface, combining video capture, image scanning, and display into a single threaded component.

<Steps>

### Step 1: Create a Processor

```c ~/workspace/source/examples/processor.c
/* create a Processor with threading enabled */
zbar_processor_t *proc = zbar_processor_create(1);
if (!proc) {
    fprintf(stderr, "Failed to create processor\n");
    return -1;
}
```

The `1` parameter enables threading. When threaded:
- Video capture runs in a background thread
- The display window runs in a separate input thread  
- Your main thread remains responsive

### Step 2: Configure symbology support

```c ~/workspace/source/examples/processor.c
/* enable all symbologies */
zbar_processor_set_config(proc, 0, ZBAR_CFG_ENABLE, 1);

/* or enable specific types */
zbar_processor_set_config(proc, ZBAR_QRCODE, ZBAR_CFG_ENABLE, 1);
zbar_processor_parse_config(proc, "code128.enable=1");
```

### Step 3: Initialize the video device

```c ~/workspace/source/examples/processor.c
const char *device = "/dev/video0";

/* initialize with video device and display enabled */
if (zbar_processor_init(proc, device, 1) != 0) {
    fprintf(stderr, "Failed to open video device\n");
    zbar_processor_error_spew(proc, 0);
    return -1;
}
```

**Device names:**
- **Linux:** `/dev/video0`, `/dev/video1`, etc.
- **Windows:** DirectShow device path
- **Empty string (`""`)**: Auto-detect default device

The second parameter (`1`) enables the display window.

### Step 4: Set up a data handler

Register a callback to receive scan results:

```c ~/workspace/source/examples/processor.c
static void my_handler(zbar_image_t *image, const void *userdata)
{
    /* extract results */
    const zbar_symbol_t *symbol = zbar_image_first_symbol(image);

    for (; symbol; symbol = zbar_symbol_next(symbol)) {
        zbar_symbol_type_t typ = zbar_symbol_get_type(symbol);
        const char *data       = zbar_symbol_get_data(symbol);

        printf("decoded %s symbol \"%s\"\n", zbar_get_symbol_name(typ), data);
    }
}

zbar_processor_set_data_handler(proc, my_handler, NULL);
```

The handler is called from the video thread whenever new symbols are decoded.

### Step 5: Start scanning

```c ~/workspace/source/examples/processor.c
/* enable the preview window */
zbar_processor_set_visible(proc, 1);

/* start scanning */
zbar_processor_set_active(proc, 1);

/* wait for user input (blocks until window closed or key pressed) */
zbar_processor_user_wait(proc, -1);
```

### Step 6: Clean up

```c ~/workspace/source/examples/processor.c
zbar_processor_destroy(proc);
```

</Steps>

## Complete Processor Example

```c ~/workspace/source/examples/processor.c
#include <stdio.h>
#include <zbar.h>

static void my_handler(zbar_image_t *image, const void *userdata)
{
    /* extract results */
    const zbar_symbol_t *symbol = zbar_image_first_symbol(image);

    for (; symbol; symbol = zbar_symbol_next(symbol)) {
        /* do something useful with results */
        zbar_symbol_type_t typ = zbar_symbol_get_type(symbol);
        const char *data       = zbar_symbol_get_data(symbol);

        printf("decoded %s symbol \"%s\"\n", zbar_get_symbol_name(typ), data);
    }
}

int main(int argc, char **argv)
{
    const char *device = "/dev/video0";

    /* create a Processor */
    zbar_processor_t *proc = zbar_processor_create(1);

    /* configure the Processor */
    zbar_processor_set_config(proc, 0, ZBAR_CFG_ENABLE, 1);

    /* initialize the Processor */
    if (argc > 1)
        device = argv[1];
    zbar_processor_init(proc, device, 1);

    /* setup a callback */
    zbar_processor_set_data_handler(proc, my_handler, NULL);

    /* enable the preview window */
    zbar_processor_set_visible(proc, 1);
    zbar_processor_set_active(proc, 1);

    /* keep scanning until user provides key/mouse input */
    zbar_processor_user_wait(proc, -1);

    /* clean up */
    zbar_processor_destroy(proc);

    return (0);
}
```

## Video Control and Configuration

### Requesting Video Size

```c
/* Request 1280x720 resolution (must be called before init) */
zbar_processor_request_size(proc, 1280, 720);
zbar_processor_init(proc, device, 1);
```

The driver may adjust or ignore this request based on hardware capabilities.

### Camera Controls

Adjust brightness, contrast, and other camera settings:

```c ~/workspace/source/zbarcam/zbarcam.c
int value;

/* Get current brightness */
zbar_processor_get_control(proc, "Brightness", &value);

/* Increase brightness */
zbar_processor_set_control(proc, "Brightness", value + 10);
```

### V4L2 Interface Options

```c
/* Request specific V4L2 interface version (Linux only) */
zbar_processor_request_interface(proc, 2);

/* Request I/O mode:
   0 = auto-detect
   1 = read()
   2 = mmap()
   3 = USERPTR (V4L2 only) */
zbar_processor_request_iomode(proc, 2);
```

## Using zbarcam

The `zbarcam` utility provides command-line access to video scanning.

### Basic Usage

```bash
# Use default video device
zbarcam

# Specify device
zbarcam /dev/video0

# Scan one barcode and exit
zbarcam --oneshot /dev/video0
```

### Output Formats

<CodeGroup>

```bash Default
zbarcam /dev/video0
# Output: CODE128:1234567890
```

```bash XML Format  
zbarcam --xml /dev/video0
# Output: Full XML with metadata
```

```bash Raw Data
zbarcam --raw /dev/video0
# Output: Just the decoded data, no type prefix
```

</CodeGroup>

### Configuration Options

```bash ~/workspace/source/zbarcam/zbarcam.c
# Disable display window
zbarcam --nodisplay /dev/video0

# Quiet mode (no beep)
zbarcam --quiet /dev/video0

# Enable only QR codes
zbarcam --set qrcode.enable=1 --set '*.enable=0' /dev/video0

# Set minimum barcode length
zbarcam --set code128.min-length=8 /dev/video0

# Request specific resolution
zbarcam --prescale=640x480 /dev/video0

# Increase verbosity
zbarcam -vvv /dev/video0
```

### Configuration String Format

The `--set` / `-S` option uses the format `[symbology.]config[=value]`:

```bash
# Enable QR codes
-S qrcode.enable=1

# Disable all symbologies
-S '*.enable=0'

# Set Code 128 minimum length
-S code128.min-length=10

# Enable test for inverted barcodes
-S '*.test-inverted=1'
```

See the [Configuration](/guides/configuration) guide for all available options.

## V4L2 Video Setup (Linux)

### Checking Available Devices

```bash
# List video devices
ls -l /dev/video*

# Check device capabilities
v4l2-ctl --list-devices

# Query device formats
v4l2-ctl -d /dev/video0 --list-formats-ext
```

### Testing Device

```bash
# Test with verbose output
zbarcam -vvv /dev/video0

# Check supported formats
zbarcam --verbose=16 /dev/video0 2>&1 | grep format
```

### Common V4L2 Issues

<Accordion title="Permission Denied">
Add your user to the `video` group:

```bash
sudo usermod -a -G video $USER
# Log out and back in for changes to take effect
```
</Accordion>

<Accordion title="Device Busy">
Another application is using the camera. Close other applications like:
- Cheese
- Skype/Zoom
- Other instances of zbarcam

```bash
# Find processes using video device
lsof /dev/video0
```
</Accordion>

<Accordion title="Unsupported Format">
ZBar will attempt format conversion automatically. If issues persist:

```bash
# Force specific format (YUYV example)
zbarcam --infmt=YUYV /dev/video0
```
</Accordion>

## Real-Time Processing Flow

The Processor uses this internal flow when threaded:

```mermaid
graph LR
    A[Video Thread] -->|Capture Frame| B[Process Image]
    B -->|Convert to Y800| C[Scan Image]
    C -->|Symbols Found| D[Call Handler]
    D --> E[Display Thread]
    E -->|Draw Results| F[Window]
    A -->|Next Frame| B
```

### Threading Model

When `zbar_processor_create(1)` enables threading:

```c ~/workspace/source/zbar/processor.c
static ZTHREAD proc_video_thread(void *arg)
{
    zbar_processor_t *proc = arg;
    zbar_thread_t *thread  = &proc->video_thread;

    while (thread->started) {
        /* wait for streaming to be active */
        while (thread->started && !proc->streaming)
            _zbar_event_wait(&thread->notify, &proc->mutex, NULL);

        /* blocking capture from video */
        img = zbar_video_next_image(proc->video);

        /* process the image */
        if (thread->started && proc->streaming)
            _zbar_process_image(proc, img);

        zbar_image_destroy(img);
    }
    return (0);
}
```

The video thread:
1. Captures frames in the background
2. Processes each frame for barcodes
3. Calls your data handler when symbols are found
4. Continues until stopped

## Performance Tips

<Tip>
**Enable Result Caching** for video streams to filter duplicate detections:

```c
zbar_image_scanner_enable_cache(proc->scanner, 1);
```

This reduces redundant callbacks for the same barcode across consecutive frames.
</Tip>

### Adjusting Scan Density

```c
/* Reduce vertical scan density for better performance */
zbar_processor_parse_config(proc, "*.y-density=1");

/* Reduce horizontal scan density */
zbar_processor_parse_config(proc, "*.x-density=1");
```

Lower density (1-2) scans fewer lines, improving speed at the cost of detection rate.

### Resolution vs Performance

- **Lower resolution** (640x480): Faster processing, works for large barcodes
- **Higher resolution** (1280x720+): Better for small or distant barcodes

Request optimal resolution before initialization:

```c
zbar_processor_request_size(proc, 640, 480);
```

## Error Handling

```c ~/workspace/source/zbarcam/zbarcam.c
if (zbar_processor_init(proc, video_device, display))
    return (zbar_processor_error_spew(proc, 0));

if (display && zbar_processor_set_visible(proc, 1))
    return (zbar_processor_error_spew(proc, 0));

if (zbar_processor_set_active(proc, 1))
    return (zbar_processor_error_spew(proc, 0));
```

The `error_spew` function prints detailed error information to stderr and returns a non-zero exit code.
